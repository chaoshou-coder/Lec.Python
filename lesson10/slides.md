## 5. 第三讲(15 分钟) —— Day01-09 串讲:高频易错集中歼灭

### 知识点 5.1 三大高频错题回顾

**错题类型 1 —— 条件分支**(`if` 写错边界)

```python
# ❌ 边界错误
if score >= 60:
    print("及格")
elif score >= 80:             # 永远执行不到
    print("良好")
```

**错题类型 2 —— 循环 + 累加器**

```python
# ❌ 累加变量在循环内初始化 → 永远只加最后一个
for price in prices:
    total = 0
    total += price
```

**错题类型 3 —— 函数 return vs print**

```python
# ❌ 把结果 print 到屏幕,其他函数拿不到
def total(prices):
    print(sum(prices))

print(total([1, 2, 3]))  # None
```

### 知识点 5.2 综合排错方法论

遇到报错三步走:
1. 看**最后一行异常类型** —— `ValueError` / `TypeError` / `IndexError`。
2. 看**报错行号** —— 定位到代码。
3. 在报错行 `print(type(变量))` + `print(变量)` 找类型不一致。

> 口诀:**报错不慌张,看类型 → 看行号 → print 类型和值**。

```python
# 参考终态(教师示范)
import json

商品库 = [
    {"id": 1, "name": "苹果", "price": 5.50},
    {"id": 2, "name": "牛奶", "price": 12.80},
    {"id": 3, "name": "面包", "price": 8.00},
]
购物车 = []

def 查找商品(编号):
    for 商品 in 商品库:
        if 商品["id"] == 编号:
            return 商品
    return None

def 浏览商品():
    for 商品 in 商品库:
        print(f"{商品['id']}. {商品['name']} ￥{商品['price']}")

def 加入购物车():
    try:
        编号 = int(input("请输入商品编号:"))
    except ValueError:
        print("输入错误!")
        return
    商品 = 查找商品(编号)
    if not 商品:
        print("商品不存在!")
        return
    for 项 in 购物车:
        if 项["id"] == 编号:
            项["qty"] += 1
            return
    购物车.append({"id": 编号, "name": 商品["name"],
                   "price": 商品["price"], "qty": 1})

def 结算():
    if not 购物车:
        print("购物车为空!")
        return
    总价 = 0
    for 项 in 购物车:
        小计 = 项["price"] * 项["qty"]
        print(f"{项['name']} x{项['qty']} = ￥{小计:.2f}")
        总价 += 小计
    print(f"总价:￥{总价:.2f}")
    购物车.clear()
```

> 而不是 bug 有没有;bug 让学生自己改 3 分钟,不改再巡到。

## 易错点

1. **`input` 永远返回字符串**,和数字比较必须 `int()`/`float()` 转。
2. **累加变量在循环外初始化**,循环内只累加,否则每次清零。
3. **函数内改列表用 `.clear()` / `append()` 等原地方法**,不要
   `lst = []` 赋值(那会创建新局部变量,原列表没变)。
4. **`return` 出去的值才能被其他函数使用**,`print` 只给人看,
   没人能接。
5. **全局变量在函数内只读不需要 `global`**,要**改**才需要声明
   `global`。
