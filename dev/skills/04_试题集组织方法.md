# 04 · 试题集组织方法 —— 如何组织三层试题集

> 用途:为每个知识点系统设计当堂练 + 课后作业 + 综合项目的三层试题集。
> 适用场景:新课程出题、现有课程题库扩充、题型难度调整。

---

## 1. 题型设计原则:⭐~⭐⭐⭐⭐⭐ 五级难度模型

| 难度 | 名称 | 典型描述 | 知识点数 | 学员动作 |
|---|---|---|---|---|
| ⭐ | 纯语法复制 | "照着示例写一遍" | 1 | 记忆 |
| ⭐⭐ | 单步变形 | "把示例的数值改一下" | 1 | 理解 |
| ⭐⭐⭐ | 组合 | "把两个知识点组合起来" | 2-3 | 应用 |
| ⭐⭐⭐⭐ | 抽象建模 | "把实际问题抽象成代码" | 3-4 | 分析 |
| ⭐⭐⭐⭐⭐ | 综合项目 | "做一个完整的小项目" | 5+ | 创造 |

### 难度分配比例

| 题型层级 | ⭐ | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
|---|---|---|---|---|---|
| 当堂练(6 道/天) | 2 | 2 | 2 | 0 | 0 |
| 课后作业(3 道/天) | 0 | 1 | 1 | 1 | 0 |
| 综合项目(1 道/周) | 0 | 0 | 0 | 0 | 1 |

**设计逻辑**:
- 当堂练以 ⭐~⭐⭐⭐ 为主,保证 80% 学员能在课上完成
- 课后作业以 ⭐⭐⭐~⭐⭐⭐⭐ 为主,需要思考,可做可不做(选做)
- 综合项目是 ⭐⭐⭐⭐⭐,需要综合多天知识,有验收标准

## 2. 当堂练(6 道/天)的设计要点

### 2.1 覆盖边界

每道题必须覆盖以下边界中的至少一个:

| 边界类型 | 示例 | 常见错误 |
|---|---|---|
| 空输入 | `[]` / `""` / `{}` | 没处理空,直接崩溃 |
| 零 | `0` / `0.0` | 除以零错误 |
| 负数 | `-1` / `-100` | 逻辑不考虑负数 |
| 单元素 | `[1]` / `"a"` | 只考虑多元素情况 |
| 重复元素 | `[1, 1, 1]` | 没考虑重复 |
| 极大/极小值 | `999999` / `-999999` | 溢出或逻辑错误 |
| 类型混合 | `[1, "a", None]` | 假设输入类型一致 |

**设计技巧**:
- ⭐ 题:只给正常输入,学员只需复制语法
- ⭐⭐ 题:给一个边界,学员需要简单处理
- ⭐⭐⭐ 题:给多个边界,学员需要组合处理

### 2.2 逐步递进

当堂练的 6 道题必须是"线性递进"的:

```
题 1:纯语法复制(⭐) → 建立信心
题 2:简单变形(⭐) → 巩固语法
题 3:加一个边界(⭐⭐) → 开始思考
题 4:组合两个知识点(⭐⭐) → 知识整合
题 5:小综合(⭐⭐⭐) → 独立思考
题 6:综合应用(⭐⭐⭐) → 挑战题(可做可不做)
```

**设计技巧**:
- 题 1-2:所有学员都应在 5 分钟内完成
- 题 3-4:中等学员应在 10 分钟内完成
- 题 5-6:优秀学员应在 15 分钟内完成
- 题 6 标注"挑战题",不强制完成

## 3. 课后作业(3 道/天)的设计要点

### 3.1 综合运用

每道课后作业必须综合运用当天 + 前一天的知识。

**为什么**:课后作业的目的是"巩固 + 联结",不是"复制课堂"。

**具体策略**:
```
Day 5 课后作业:
  题 1(Day 5 知识):字符串 replace 应用 → ⭐⭐⭐
  题 2(Day 5 + Day 4):字符串 + 循环 → ⭐⭐⭐
  题 3(Day 5 + Day 4 + Day 3):字符串 + 循环 + 分支 → ⭐⭐⭐⭐
```

### 3.2 可做可不做(选做)

课后作业标注为"选做",不强制完成。

**为什么**:零基础学员的课外时间不确定,强制作业会导致"抄答案"。
选做的目的是让有能力的学员挑战,不打击能力弱的学员。

**标注方式**:
```python
"""
[难度: ⭐⭐⭐⭐]
[所属知识点: 字符串 + 循环 + 分支]
[预计完成时间: 30 分钟]
[类型: 选做] ← 标注选做
"""
```

## 4. 综合项目的设计要点

### 4.1 真实场景

综合项目必须是真实场景,不是"计算 1+2+...+100"。

**真实场景示例**:
- Day 7:购物车程序(日常购物)
- Day 14:日志写入器(日常记录)
- Day 25:电影评分预测(数据分析)
- Day 36:聊天机器人(NLP 应用)
- Day 43:微调一个问答模型(LLM 应用)
- Day 50:爬取豆瓣电影 Top 250(数据采集)
- Day 60:AI 知识库(综合应用)

**为什么**:真实场景让学员"看到价值",提高学习动力。

### 4.2 多知识点串联

每个综合项目必须串联至少 5 个知识点。

**示例 —— Day 36 聊天机器人**:
```
知识点串联:
1. 函数(def) → 封装对话逻辑
2. 字典(dict) → 存储问答对
3. 循环(while) → 持续对话
4. 条件分支(if) → 匹配问题
5. 字符串方法(split/join) → 处理输入
6. 文件读写(optional) → 保存对话历史
```

### 4.3 验收标准

每个综合项目必须有明确的验收标准(Rubric)。

**示例 —— Day 7 购物车程序验收标准**:
```
✅ 能添加商品到购物车(10 分)
✅ 能从购物车移除商品(10 分)
✅ 能计算总价(10 分)
✅ 能显示购物车内容(10 分)
✅ 能处理异常输入(10 分)
✅ 代码结构清晰(函数封装)(10 分)
✅ 有 README 说明(10 分)
✅ 有 demo 演示(10 分)
✅ 扩展功能(折扣/库存/历史)(10 分)
✅ 创新功能(自己设计)(10 分)
总分:100 分,60 分及格,80 分良好,90 分优秀
```

## 6. 出题流程

### 6.1 单题出题流程

```
Step 1:确定题目所属知识点
Step 2:确定题目难度(⭐~⭐⭐⭐⭐⭐)
Step 3:设计场景(真实场景优先)
Step 4:编写题目描述(≤ 100 字符/行)
Step 5:编写示例(至少 1 个正常 + 1 个边界)
Step 6:编写学员代码区(pass 占位)
Step 7:编写测试区(≥ 2 个测试)
Step 8:教师自己写一遍答案,验证可行性
Step 9:找一位学员试做,记录耗时
Step 10:根据反馈调整难度/描述
```

### 6.2 每日出题流程

```
Step 1:确定本天知识点(≤ 2 个新概念)
Step 2:设计 6 道当堂练(⭐~⭐⭐⭐,线性递进)
Step 3:设计 3 道课后作业(⭐⭐~⭐⭐⭐⭐,综合运用)
Step 4:检查覆盖边界(空输入/零/负数/单元素)
Step 5:编写参考答案
Step 6:教师自己做一遍,记录耗时
Step 7:调整难度(如果教师自己做 > 15 分钟,降低难度)
```

### 6.3 综合项目出题流程

```
Step 1:确定项目主题(真实场景)
Step 2:列出串联的知识点(≥ 5 个)
Step 3:设计验收标准(Rubric,10 个维度)
Step 4:编写项目描述(背景/需求/验收标准)
Step 5:编写"起步代码"(部分 scaffolding)
Step 6:教师自己做一遍,记录耗时
Step 7:找学员试做,调整难度
Step 8:编写参考答案 + 扩展思路
```

## 7. 常见出题陷阱

| 陷阱 | 案例 | 后果 | 避免方式 |
|---|---|---|---|
| 难度跳跃 | 题 1 是 ⭐,题 2 是 ⭐⭐⭐⭐ | 学员卡住 | 线性递进,不跳级 |
| 场景虚假 | "计算 1+2+...+100" | 学员无动力 | 用真实场景 |
| 边界遗漏 | 只给正常输入 | 学员代码脆弱 | 必须覆盖 7 大边界 |
| 描述不清 | "请实现相应功能" | 学员不知道做什么 | 写清输入/输出/约束 |
| 答案错误 | 教师提供的答案有 bug | 学员被误导 | 教师先自己做一遍 |
| 题量过多 | 10 道当堂练 | 学员做不完 | 严格 6 道,题 6 选做 |
