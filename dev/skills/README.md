## 使用方式

- **新开一门课**:先看 `01` 推终点 → `02` 调研参照 → `03` 编排顺序 → `04` 设计题目 → `05` 排版交付
- **迭代现有课**:直接跳到对应 skill 查决策依据,避免重复踩坑
- **新人上手**:通读 5 个文件后,能独立承担一天的课程开发

## 维护原则

1. **每次迭代必更新**:课程结构调整、题型增删、排版微调,都要回写对应 skill
2. **决策留痕**:任何"为什么这么排"的讨论结论,写入 `03` 或 `04` 的决策记录
3. **数据可验证**:调研数据保留来源链接,题型标注来源课程
4. **格式一致**:所有 .md 文件 ≤ 100 字符/行,表格优先于大段文字
