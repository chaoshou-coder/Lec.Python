# 01 · 知识地图推理 —— 如何从零推导出终点能力所需的全部知识点

> 用途:给定一个"终点能力"(如"微调 LLM"),系统化地倒推出完整的知识链条,
> 避免"跳阶"和"遗漏前置"两大常见陷阱。
> 适用场景:新课程设计、某一模块的知识点审计、学员诊断(哪块基础弱)。

---

## 1. 方法论:从终点倒推知识链条

### 1.1 推理三步法

```
终点能力 → 第一层前置 → 第二层前置 → ... → 原子知识点(已掌握的起点)
```

**Step 1 · 写出终点能力的直接前置**
问自己:"要掌握 X,必须已经会什么?"列出所有直接前置。

示例 —— 终点能力"微调 LLM":
- 要微调 LLM → 需要理解 **LoRA / QLoRA** 的原理
- 要微调 LLM → 需要掌握 **PyTorch 训练循环**
- 要微调 LLM → 需要理解 **Tokenizer + Attention Mask**
- 要微调 LLM → 需要能从 HuggingFace 加载预训练模型

**Step 2 · 递归展开每个前置**
对每个前置,重复问"要掌握它,必须已经会什么",直到触及学员已掌握的原子知识。

以"LoRA 原理"为例:
- LoRA → 理解矩阵低秩分解 → 理解矩阵乘法 → 理解 **NumPy / 张量运算**
- LoRA → 理解微调 vs 全量微调 → 理解 **梯度下降 / 反向传播**
- LoRA → 理解 HuggingFace PEFT 库 → 理解 **Python 类与对象**

**Step 3 · 合并、去重、拓扑排序**
把所有"展开后的节点 + 边"合并成有向无环图(DAG),做拓扑排序 → 得到学习顺序。

### 1.2 为什么倒推而不是顺推?

顺推(从 Python print 开始一点点讲)的问题:
- **不知道为什么要学**:学员学到函数时不知道这跟 AI 有什么关系
- **动力衰减**:60 天时,前 20 天还没见到 AI,流失率高
- **冗余**:顺推会把"永远用不到的知识"也塞进去

倒推的优势:
- **目标清晰**:每个知识点都能讲清楚"学这个是为了..."
- **最小必要**:只教终点能力真正需要的,不贪全
- **支撑课程压缩**:21 天→10 天压缩的关键就是"倒推后只保留必要链"

### 1.3 知识依赖图的构建方法

#### 节点定义
每个节点是一个"知识点",粒度参考:
- ✅ "列表 CRUD" (具体到操作层面)
- ✅ "梯度下降原理" (具体到理解层面)
- ❌ "Python 基础" (太粗,不可执行)
- ❌ "会写 for 循环" (太细,可合并到"控制流")

#### 边定义
A → B 表示"必须先掌握 A,才能学 B"。
三种依赖强度:
- **强依赖(实线)**:不会 A 完全无法学 B (例:不会矩阵乘法 → 无法学 Attention)
- **弱依赖(虚线)**:不会 A 也能学 B,但理解不深 (例:不会 Vim → 也能学 Python)
- **可同步(Dashed)**:两节课放在同一天的不同时段,互相不依赖

#### 拓扑排序算法
```
1. 找到所有"入度为 0"的节点(没有前置) → 作为第 1 天
2. 移除这些节点,重新计算入度
3. 重复,直到所有节点被排序
4. 若剩余节点入度都 > 0 → 说明有环,需要检查依赖定义
```

**实操建议**:68 个节点用手画太累,用 Excel 或 Obsidian 画 DAG 即可,不需要专业工具。

## 2. 每个知识点的"最低要求"标注

不是所有知识点都需要"深入掌握"。对每个节点标注以下三类之一:

| 标注 | 含义 | 教学策略 | 示例 |
|---|---|---|---|
| **课前掌握** | 必须在进入该节点前 100% 会 | 第一天测查,不过关补课 | NumPy 切片(学 Attention 前必须会) |
| **可同步学习** | 本节课同步教,学员可接受 | 课上直接讲,配足练习 | 张量概念(学矩阵乘法时同步引入) |
| **可课后深入** | 能理解主干就行,细节可后续补 | 标注"进阶链接",不深讲 | 矩阵微积分推导(理解梯度下降即可) |

### 如何判断属于哪一类?

问两个问题:
1. **前置节点的输出,是否直接作为当前节点的输入?**
   - 是 → 课前掌握
   - 否,只是辅助理解 → 可同步学习
2. **当前节点的细节,是否影响后续节点的学习?**
   - 是 → 不能只给"可同步",需要课前掌握
   - 否 → 可课后深入

示例:
- "矩阵乘法"是"Attention"的**课前掌握**,因为 Q·K^T 全靠它
- "Softmax 的数值稳定性"是"Attention"的**可课后深入**,因为初学者只会调用就行
- "Transformer 架构全貌"是"LoRA"的**可同步学习**,因为讲 LoRA 时需要回顾

## 4. 常见陷阱

### 4.1 跳阶(还没教 A 就教 B)

| 跳阶案例 | 后果 | 修复方式 |
|---|---|---|
| 还没教循环就教递归 | 学员完全无法理解递归终止条件 | 递归必须放在循环之后 |
| 还没教矩阵乘法就教 Attention | 学员看不懂 Q·K^T 在做什么 | 先补 NumPy 矩阵运算 |
| 还没教函数就教 nn.Module | 看不懂 class 和 __init__ | 先教 def 再教 class |
| 还没教 JSON 就教数据集加载 | 看不懂 data.json 结构 | 先教 JSON 读写 |

**检测方法**:对每个新知识点,检查其所有前置是否都在"已学"集合中。

### 4.2 遗漏前置(学 B 时默认学员会 A)

| 遗漏案例 | 后果 | 修复方式 |
|---|---|---|
| 学 Attention 默认会矩阵乘法 | 学员卡住 | 课前测查 + 补课 |
| 学 LoRA 默认会 PyTorch | 学员看不懂 nn.Linear | 先教 PyTorch 基础 |
| 学爬虫默认会 HTTP 协议 | 看不懂状态码 | 先讲 HTTP 基础 |
| 学 RAG 默认会向量嵌入 | 看不懂 embedding | 先讲 Word2Vec |

**检测方法**:对每个新知识点,问"学员可能在哪里卡住",列出所有"默认会"的假设。

### 4.3 冗余(教了不需要的知识)

| 冗余案例 | 后果 | 修复方式 |
|---|---|---|
| 花 1 天讲 Python 装饰器 | 对 AI 方向几乎不用 | 移到"可课后深入" |
| 详细讲矩阵微积分推导 | 学员用 autograd 就行 | 只讲直觉,不推导 |
| 讲 10 种排序算法 | 实际只用 sorted | 只讲 sorted 用法 |
| 讲元类/描述符 | 对 AI 方向无用 | 删除 |

**检测方法**:对每个知识点,问"这个知识在终点能力中用到吗?" → 不是就不教。

## 6. 知识地图的更新机制

知识地图不是一次性产物,需要持续迭代:

1. **每次学员卡住**:检查是否遗漏前置 → 补边
2. **每次课程压缩**:检查是否有冗余节点 → 删节点
3. **每次技术更新**:检查是否有新节点需要加入 → 增节点
4. **每轮教学后**:根据学员反馈调整"最低要求"标注

**版本管理**:建议用 Git 管理知识地图,每次修改写清楚"为什么改"。
